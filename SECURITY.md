# Security Policy

## Responsible Disclosure
Se identifichi una vulnerabilità, non aprire issue pubbliche.

Invia una segnalazione privata ai maintainer includendo:
- Descrizione del problema
- Impatto atteso
- Passi per riprodurre
- Eventuale proof-of-concept minimale

## Scope
Rientrano nel perimetro prioritario:
- Cifratura dati sensibili
- Backup/restore
- Export referti
- Controllo accessi locali (lock app)

## Gestione segnalazioni
Obiettivo operativo:
- Presa in carico iniziale: entro 7 giorni
- Aggiornamenti periodici fino a mitigazione/fix

## Buone pratiche per contributori
- Non inserire PHI reali in issue, PR, log o screenshot
- Usa dataset sintetici per test e bug report
- Evita output diagnostici con dati clinici in chiaro
